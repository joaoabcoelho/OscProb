1 # OscProb {#index}
2 
3 OscProb is a small set of classes aimed at computing exact neutrino oscillation probabilities with a few different models.
4 
5 OscProb contains a basic framework for computing neutrino oscillation probabilities. It is integrated into [ROOT](https://root.cern.ch/), so that each class can be used as you would any ROOT class.
6 
7 Available classes are:
8 - **[PremModel](@ref OscProb::PremModel):** Used for determining neutrino paths through the earth
9 - **[PMNS_Fast](@ref OscProb::PMNS_Fast):** Standard 3-flavour oscillations
10 - **[PMNS_Iter](@ref OscProb::PMNS_Iter):** Standard 3-flavour oscillations (iterative)
11 - **[PMNS_Sterile](@ref OscProb::PMNS_Sterile):** Oscillations with any number of neutrinos
12 - **[PMNS_NSI](@ref OscProb::PMNS_NSI):** Oscillations with 3 flavours including Non-Standard Interactions
13 - **[PMNS_Deco](@ref OscProb::PMNS_Deco):** Oscillations with 3 flavours including a simple decoherence model
14 - **[PMNS_LIV](@ref OscProb::PMNS_LIV):** Oscillations with 3 flavours including Lorentz Invariance Violations
15 - **[PMNS_Decay](@ref OscProb::PMNS_Decay):** Oscillations with 3 flavours including neutrino decays of the second and third neutrino mass states nu_2 and nu_3. [Requires external library Eigen3, see the instructions below.]
16 - **[Absorption](@ref OscProb::Absorption):** Computes absorption probabilities for high-energy neutrinos
17 
18 A few example macros on how to use OscProb are available in a tutorial directory.

20 # Installing OscProb
21 
22 OscProb is very easy to install. The only requirements is to have ROOT installed with the GSL libraries.
23 
24 **NEW: Thanks to Jacek Holeczek, OscProb now also builds with ROOT 6!!**
25 
26 In order to compile the [PMNS_Decay](@ref OscProb::PMNS_Decay) class, it is necessary to donwload the external Eigen library. This library is added as a submodule. There are two options:
27 - During cloning: `git clone --recurse-submodules https://github.com/joaoabcoelho/OscProb.git`
28 - After clonning: `git submodule update --init`
29 
30 Once you have ROOT setup, simply do:
31 ```sh
32 cd OscProb
33 make
34 ```
35 
36 A shared library will be produced: ```libOscProb.so```
37 
38 This should take a few seconds and you are all set.
39 
40 Just load the shared library in your ROOT macros with:
41 ```cpp
42 gSystem->Load("/full/path/to/libOscProb.so");
43 ```
44 
45 Or use the ```LoadOscProb.C``` macro (see below).

47 # Tutorial
48 
49 In the directory OscProb/tutorial you will find a few macros with examples using OscProb.
50 
51 Two macros are particularly useful:
52 - ```simpleExamples.C``` : Contains some short pieces of code on how to perform different tasks.
53 - ```MakeOscillogram.C``` : Runs a full example of how to plot an oscillogram with the PREM model.
54 
55 Additionally, these macros contain useful tools:
56 - ```LoadOscProb.C```: Searches for the OscProb library in your current directory, parent directory, or library path, and then loads it. It is called within the tutorial macros as a possible usage example.
57 - ```SetNiceStyle.C```: Provides simple tools to make your plots look nicer. Feel free to use it anytime you're making plots, even if you're not running OscProb. This is completely independent of OscProb.
58 
59 To run macros in compiled mode you will need to preload the OscProb library, e.g:
60 
61 ```sh
62 root -l LoadOscProb.C MakeOscillogram.C+
63 ```
